# Инструкции по управлению Memory Bank

Этот документ описывает базовые правила сопровождения Memory Bank в текущем репозитории.

## Назначение

* Поддерживать единый источник проектного контекста в каталоге `memory-bank/`.
* Обеспечивать предсказуемое обновление статуса, решений, прогресса и паттернов.
* Сохранять сведения об окружении в отдельном файле для конкретного hostname.

## Правила редактирования

* Все записи ведутся на русском языке.
* Все временные метки указываются в UTC в формате ГГГГ-ММ-ДД ЧЧ:ММ:СС.
* В рабочих разделах новые записи добавляются сверху, если явно не указано иное.
* `productContext.md` хранит сноски обновлений в конце файла.
* `system_full_info_<hostname>.md` должен соответствовать текущему хосту и не должен перезаписывать паспорт другой машины.

## Состав Memory Bank

* `memory-bank/activeContext.md` — текущий фокус, недавние изменения и открытые вопросы.
* `memory-bank/decisionLog.md` — архитектурные и реализационные решения.
* `memory-bank/productContext.md` — цели продукта, ключевые возможности и архитектура.
* `memory-bank/progress.md` — выполненные, текущие и следующие задачи.
* `memory-bank/systemPatterns.md` — устойчивые паттерны кодирования, архитектуры и тестирования.
* `memory-bank/system_full_info_x200.md` — паспорт окружения текущего хоста.

## Базовый процесс обновления

### Обновление контекста

1. Зафиксировать изменение в соответствующем файле каталога `memory-bank/`.
2. Добавить запись с корректной временной меткой UTC.
3. Разместить новую запись в начале рабочего раздела.

### Обновление паспорта окружения

1. Проверить актуальный hostname.
2. Обновлять только соответствующий файл `system_full_info_<hostname>.md`.
3. Добавить изменение в `## Change Log` и затем обновить профильные секции.

### Контроль качества

1. Проверить, что структура `memory-bank/` остаётся полной.
2. Убедиться, что шаблонные разделы не удалены.
3. Поддерживать единый формат записей и формулировок.
