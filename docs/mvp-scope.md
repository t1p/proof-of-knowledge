## Семантика результата валидации
- Результат PoK в MVP трактуется как: **«объект соответствует протоколу X сообщества Y»**.
- Результат PoK в MVP **не** трактуется как «абсолютная/универсальная истина».

## Пилотный домен MVP (фиксированный)
- **Один домен для MVP:** документы/заявления и публичные утверждения в экосистеме МТЛ/Монтелиберо.
- Причина фиксации: в текущем контексте репозитория нет более явно приоритетного и лучше формализованного пилотного домена.

## Единица валидации
- Валидируемый объект: артефакт любого размера (текст, документ, страница, набор данных, медиа-снимок и т.д.).
- Для каждого объекта обязателен **claim list**: явный список проверяемых утверждений.
- Итог валидации формируется по claim list в рамках протокола выбранного сообщества.

## Роли MVP
- **Applicant** — подает объект и claim list на валидацию.
- **Validator** — проверяет claims по опубликованному протоколу и выносит заключение.
- **Community** — определяет протокол/правила, принимает валидаторов, хранит историю решений.
- **Observer** — наблюдает статусы и историю валидаций без участия в принятии решения.

## Минимальный happy-path (6 шагов)
1. Создать сообщество.
2. Опубликовать протокол валидации сообщества.
3. Вступить валидатору в сообщество.
4. Подать заявку (объект + claim list).
5. Провалидировать заявку валидатором(ами) по протоколу.
6. Финализировать результат со статусом соответствия/несоответствия протоколу.

## Анти-мусор / анти-сговор (MVP-уровень)
- [`deposit/fee`](docs/mvp-scope.md:1): заявка и участие в валидации требуют минимального экономического порога для снижения спама.
- [`disputes-lite`](docs/mvp-scope.md:1): упрощенный challenge-механизм с **challenge deposit** для оспаривания сомнительных финализаций.
- Санкции на MVP-уровне: **freeze / slash / exile** по факту подтвержденного злоупотребления, без сложной судебной процедуры.

## Метрики MVP и gate-пороги («MVP состоялся»)
- **Cycle time (медиана «заявка → финал»):** < 72 часов.
- **Completion rate:** > 70%.
- **Dispute rate:** измеряется, без порога на MVP.
- **Validator retention:** ≥ 50% валидаторов выполнили ≥ 5 проверок.
- **User value:** средняя оценка полезности ≥ 4/5.

## Что не входит в MVP
- Атомарная семантика «по умолчанию» для всех доменов.
- Сложные веса/рейтинги экспертов и многофакторные модели доверия.
- Сложная токеномика и продвинутые экономические контуры.
- Полнофункциональные апелляции/суды и многоступенчатое арбитражное производство.
- Претензия на «универсальную истину» вне протокола конкретного сообщества.
- Production-уровень KYC.
